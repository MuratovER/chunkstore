# S3 backend

chunkstore stores chunk blobs and JSON metadata in a single S3 bucket (or MinIO / LocalStack) under a configurable **prefix**.

## Key layout

With `prefix="chunks"` (default):

| Object key | Content |
|------------|---------|
| `chunks/{digest}` | Raw chunk bytes (`digest` = 64-char SHA-256 hex) |
| `chunks/_manifest/{file_id}` | JSON manifest: `{"digests": [...], "file_bytes": N}` |
| `chunks/_refcount/{digest}` | JSON unsigned integer refcount |
| `chunks/_manifest/__index__` | JSON array of file IDs |
| `chunks/_refcount/__index__` | JSON array of digests |

Metadata lives **under the same prefix** as chunk blobs. IAM policies must allow the full prefix tree, not only digest keys.

## IAM policy (minimal)

Replace `my-bucket` and `chunks` with your bucket and prefix:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:HeadObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::my-bucket",
        "arn:aws:s3:::my-bucket/chunks/*"
      ]
    }
  ]
}
```

`ListBucket` is optional for normal ingest/read/delete; required for `ListChunkKeys` / future FSCK utilities.

## Python quickstart

```bash
pip install "chunkstore[s3]"
```

```python
from chunkstore import ChunkStore, S3Backend

backend = S3Backend(
    "my-bucket",
    prefix="chunks",
    endpoint_url="https://s3.amazonaws.com",  # omit for AWS; set for MinIO
    region_name="us-east-1",
)
store = ChunkStore.open(backend)
store.ingest("doc_v1", b"hello")
print(store.stats())
```

Retries and timeouts use botocore defaults: connect 10s, read 60s, 3 adaptive retries (override via constructor kwargs).

## Go quickstart

Build the Rust core first (see [go/README.md](../go/README.md)):

```bash
./scripts/build-core.sh
```

```go
import "github.com/MuratovER/chunkstore/go/chunkstore"

store, err := chunkstore.OpenS3(chunkstore.S3Options{
    Bucket:      "my-bucket",
    Prefix:      "chunks",
    EndpointURL: "http://localhost:9000", // MinIO
})
defer store.Close()
digests, err := store.Ingest("doc_v1", []byte("hello"))
```

## MinIO (local dev)

```bash
docker run -d -p 9000:9000 \
  -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin \
  minio/minio server /data

export AWS_ACCESS_KEY_ID=minioadmin AWS_SECRET_ACCESS_KEY=minioadmin
export AWS_DEFAULT_REGION=us-east-1
export S3_ENDPOINT_URL=http://localhost:9000 S3_BUCKET=chunkstore-test
```

- Python: `pytest -m s3`
- Go: `go test -v -tags s3` (from `go/chunkstore`)

## Cross-language compatibility

Python and Go share the same on-disk key layout. Write via Python `S3Backend`, read/delete via Go `OpenS3` on the same bucket + prefix (and vice versa). See [CONTRIBUTING.md](../CONTRIBUTING.md#on-disk-format-do-not-break-silently).
